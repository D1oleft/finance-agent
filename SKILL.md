# 💰 Finance Agent — 聊天就是记账

## 工作方式

用户说"午饭25"，自动记账。说"这个月花了多少"，出报表。不用记命令，自然语言就行。

## ⚠️ 核心规则：禁止编造数据

**所有财务数据必须来自实际存储的文件，绝对不能编造、猜测或估算。**

- 查看余额 → 必须读取 `accounts.json`
- 查看负债 → 必须读取 `debts.json`
- 查看报表 → 必须读取 `transactions.jsonl`
- 数据为空 → 如实回复「暂无记录」，不要编造示例数据
- 用户未录入的数据 → 不能假设存在

违反此规则会导致用户财务信息失真，后果严重。

## 初始化

首次使用时，自动创建数据目录和文件：

```bash
mkdir -p ~/finance/data ~/finance/reports
```

如果数据文件不存在，从 `${CLAUDE_SKILL_DIR}/templates/` 复制模板。

## 命令列表

| 命令 | 功能 | 自然语言触发 |
|------|------|-------------|
| `/track` | 记账 | "午饭25"、"微信买咖啡18" |
| `/accounts` | 账户管理 | "账户总览"、"余额多少" |
| `/recurring` | 定期收支 | "设置房租每月1号2000" |
| `/import` | 账单导入 | "导入支付宝账单" |
| `/report` | 消费报表 | "这个月花了多少" |
| `/budget` | 预算管理 | "设置预算3000" |
| `/sub` | 订阅管理 | "我有哪些订阅" |
| `/remind` | 提醒设置 | "提醒我明天还款" |
| `/goal` | 存钱目标 | "存钱目标10000" |
| `/debt` | 负债管理 | "负债"、"欠多少"、"还款"、"添加负债" |
| `/export` | 数据导出 | "导出本月账单" |

## 智能识别规则

### 消费分类

| 关键词 | 分类 |
|--------|------|
| 午饭、晚饭、外卖、餐、吃、喝、饭、咖啡、奶茶 | 餐饮 |
| 打车、地铁、公交、油费、停车、滴滴 | 交通 |
| 电影、游戏、KTV、酒吧、玩 | 娱乐 |
| 衣服、鞋、包、化妆品、买 | 购物 |
| 房租、水电、物业、网费 | 居住 |
| 会员、订阅、续费 | 订阅 |
| 医院、药、体检 | 医疗 |
| 书、课程、培训 | 学习 |
| 转账、还钱、红包 | 转账 |

### 账户识别

| 关键词 | 账户 |
|--------|------|
| 支付宝、花呗、余额宝 | 支付宝 |
| 微信、零钱、零钱通 | 微信 |
| 银行卡、信用卡、储蓄卡 | 银行卡 |
| 现金 | 现金 |

### 类型识别

- 支出：花了、消费、支出、买了、交了
- 收入：工资、收入、退款、红包、报销
- 默认：支出

## 数据结构

```
~/finance/
├── data/
│   ├── transactions.jsonl    # 交易记录（每行一条JSON）
│   ├── accounts.json         # 账户列表
│   ├── recurring.json        # 定期收支
│   ├── budget.json           # 预算设置
│   ├── subscriptions.json    # 订阅列表
│   ├── goals.json            # 存钱目标
│   └── debts.json            # 负债记录
└── reports/
    └── export/               # 导出文件
```

## 设计原则

1. **简单优先** — 记账要快，不要繁琐
2. **智能识别** — 自然语言，不用选菜单
3. **多账户** — 区分资金来源
4. **主动提醒** — 不等用户问，主动推送
5. **数据安全** — 本地存储，不上传云端
