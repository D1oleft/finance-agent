# 错误处理

## 首次使用

如果数据文件不存在：
1. 自动创建 `~/finance/data/` 目录
2. 从 `${CLAUDE_SKILL_DIR}/templates/` 复制模板文件
3. 提示用户设置账户和预算

```
👋 首次使用，已初始化数据目录

建议先设置：
1. 账户余额（/accounts 设置 支付宝 2500）
2. 月度预算（/budget 设置 3000）
```

## 金额无法识别

```
用户：吃饭了
Agent：请告诉我花了多少钱？例如：午饭25
```

## 多笔消费

```
用户：午饭25，咖啡18
Agent：请分开记录：
1. 午饭25
2. 咖啡18
```

## 账户不存在

```
用户：花呗买衣服200
Agent：花呗账户不存在。要添加吗？（回复"添加花呗"）
```

## 重复导入

导入账单时检测重复：
- 比对：时间 + 金额 + 描述
- 完全相同：跳过，提示"跳过重复 X 笔"
- 时间相同金额不同：提示确认

## 文件损坏

如果 JSON 文件格式错误：
1. 先备份损坏的文件（重命名为 `{文件名}.bak.{时间戳}`）
2. 从模板重置为默认值
3. 提示用户检查历史交易数据

```
⚠️ accounts.json 格式错误

已备份损坏文件为 accounts.json.bak.20260617120000
已重置为默认值

请检查：
1. 历史交易记录是否需要重新核对账户余额
2. 备份文件中是否有需要恢复的数据
```

## JSONL 文件损坏

如果 `transactions.jsonl` 中存在损坏的行：
1. 逐行读取，跳过无法解析的行
2. 将损坏的行写入 `{文件名}.corrupt.{时间戳}` 备份文件
3. 用有效行重新生成 `transactions.jsonl`
4. 提示用户损坏的行数

```
⚠️ transactions.jsonl 中发现 3 行损坏数据

已备份损坏数据为 transactions.jsonl.corrupt.20260617120000
已保留 125 行有效数据

请检查备份文件确认是否有重要数据丢失
```

## 预算超支

```
🚨 预算超支预警

购物分类已超预算！
预算：800元
已花：920元
超支：120元

建议：
1. 本月不再购物
2. 从其他分类借调预算
```

## 余额不足

**消费/转账时余额不足**：
```
⚠️ 余额不足

微信余额：50元
本次支出：100元

建议：
1. 切换其他账户
2. 先充值再消费
```

**还款时余额不足**：
```
⚠️ 余额不足

支付宝余额：800元
本次还款：1,000元

无法完成还款，请先充值或切换其他账户。
```

**存入目标时余额不足**：
```
⚠️ 余额不足

支付宝余额：200元
本次存入：500元

无法完成存入，请先充值或减少存入金额。
```
