# 负债管理

## 流程

### 1. 初始化检查
检查 `~/finance/data/debts.json` 是否存在，不存在则创建默认配置。

### 2. 操作类型

**查看负债总览（默认）**：
```
💳 负债总览

花呗：欠 3,200元（还款日：每月9号）
信用卡：欠 8,500元（还款日：每月25号）
借呗：欠 12,000元（还款日：每月15号）

总负债：23,700元
本月已还：2,000元
本月待还：5,500元
```

**添加负债**：
```
/debt 添加 花呗 3200 每月9号
→ 已添加负债：花呗，欠款 3,200元，还款日：每月9号

/debt 添加 信用卡 8500 每月25号
→ 已添加负债：信用卡，欠款 8,500元，还款日：每月25号
```

> **参数语义**：添加负债时指定的金额同时作为 `initial_amount`（初始欠款）和 `current_amount`（当前欠款），即添加时两者相等。

**记录还款**：
```
/debt 还款 花呗 1000
→ 花呗已还款 1,000元，剩余欠款 2,200元

/debt 还款 信用卡 5000
→ 信用卡已还款 5,000元，剩余欠款 3,500元
```

> **重要**：还款操作必须同时执行以下三步：
> 1. 更新 `debts.json` 中的 `current_amount` 和 `repayment_history`
> 2. 在 `transactions.jsonl` 中写入一笔支出交易（type: `expense`，category: `转账`）
> 3. 更新 `accounts.json` 中对应账户的余额（扣除还款金额）

**负债详情**：
```
/debt 详情 花呗
→ 花呗详情：
  - 初始欠款：5,000元
  - 已还总额：1,800元
  - 剩余欠款：3,200元
  - 还款日：每月9号
  - 还款记录：
    - 2026-06-01：还 1,000元
    - 2026-05-09：还 800元
```

**删除负债**：
```
/debt 删除 花呗
→ 已删除负债：花呗（已还清）
```

## 负债类型

| 类型 | 说明 |
|------|------|
| credit-card | 信用卡 |
| huabei | 花呗 |
| jiebei | 借呗 |
| loan | 贷款（房贷、车贷等） |
| other | 其他 |

## 数据格式

负债数据存储在 `~/finance/data/debts.json`：

```json
{
  "debts": [
    {
      "name": "花呗",
      "type": "huabei",
      "initial_amount": 5000,
      "current_amount": 3200,
      "repayment_day": "每月9号",
      "repayment_history": [
        {
          "date": "2026-06-01",
          "amount": 1000,
          "note": ""
        }
      ],
      "created_at": "2026-05-01T00:00:00+08:00"
    }
  ]
}
```

## 负债状态

| 状态 | 说明 |
|------|------|
| active | 进行中（current_amount > 0） |
| settled | 已结清（current_amount = 0） |

当负债还清后，状态标记为 `settled`，保留记录不删除，便于查看历史还款记录。

## 边界情况

- 负债不存在 → 提示未找到，询问是否添加
- 还款金额超过欠款 → 提示超额，询问是否还清
- 重复添加同名负债 → 提示已存在
- 欠款为0 → 提示已还清，状态改为 `settled`，保留记录

## 智能识别

### 金额识别
- "花呗欠3200" → 3200元
- "还了1000" → 1000元
- "信用卡8500" → 8500元

### 还款日识别
- "每月9号" → 每月9号
- "每月25号" → 每月25号
- "月底" → 每月最后一天
