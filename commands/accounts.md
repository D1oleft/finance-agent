# 账户管理

## 流程

### 1. 初始化检查
检查 `~/finance/data/accounts.json` 是否存在，不存在则创建默认配置。

### 2. 操作类型

**查看（默认）**：
```
💳 账户总览

支付宝：2,500元
微信：1,800元
银行卡：15,000元
现金：300元

总资产：19,600元
```

**添加账户**：
```
/accounts 添加 信用卡
→ 已添加账户：信用卡（类型：银行卡）
```

**设置余额**：
```
/accounts 支付宝 设置余额 5000
→ 支付宝余额已设置为 5,000元
```

**资产分析**：
```
📊 资产分析

总资产：19,600元
本月收入：8,000元
本月支出：3,200元
本月结余：4,800元
```

## 账户类型

| 类型 | 说明 |
|------|------|
| e-wallet | 电子钱包（支付宝、微信） |
| bank | 银行卡（储蓄卡） |
| cash | 现金 |
| investment | 投资账户 |
| credit | 信用账户（花呗、信用卡、借呗） |

> **信用账户说明**：信用账户（花呗、信用卡、借呗）应同时在 `accounts.json` 和 `debts.json` 中记录。消费时从信用账户扣款，同时在负债中增加相应金额。

## 边界情况

- 账户不存在 → 询问是否添加
- 余额为负 → 提示透支
- 重复添加 → 提示已存在
