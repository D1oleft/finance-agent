# 账户管理

## 流程

### 1. 初始化检查
检查 `~/finance/data/accounts.json` 是否存在，不存在则执行统一初始化流程（参见 SKILL.md 初始化章节）。

### 2. 操作类型

**查看（默认）**：

信用账户（`credit` 类型）显示负债关联信息，包括关联的负债名称和当前欠款。

```
💳 账户总览

支付宝：2,500元
微信：1,800元
银行卡：15,000元
现金：300元
花呗：-3,200元（关联负债：花呗，欠款 3,200元）

总资产（不含信用账户）：19,600元
```

**添加账户**：
```
/accounts 添加 信用卡
→ 已添加账户：信用卡（类型：信用账户(credit)）
```

**设置余额**：
```
/accounts 支付宝 设置余额 5000
→ 支付宝余额已设置为 5,000元
```

**资产分析**：

流程步骤：
1. 读取 `accounts.json`，汇总所有账户余额，计算总资产（排除 `credit` 类型的信用账户）
2. 读取 `transactions.jsonl`，筛选本月记录，按 `type` 汇总收入和支出
3. 计算本月结余 = 本月收入 - 本月支出
4. 输出分析报告

```
📊 资产分析

总资产：19,600元
本月收入：8,000元
本月支出：3,200元
本月结余：4,800元
```

**删除账户**：
```
/accounts 删除 现金
→ 已删除账户：现金
```

> **删除限制**：有关联交易记录或负债的账户不能删除，需先清除关联交易或还清负债后再删除。

## 账户类型

| 类型 | 说明 |
|------|------|
| e-wallet | 电子钱包（支付宝、微信） |
| bank | 银行卡（储蓄卡） |
| cash | 现金 |
| investment | 投资账户 |
| credit | 信用账户（花呗、信用卡、借呗） |

> **信用账户说明**：信用账户（花呗、信用卡、借呗）应同时在 `accounts.json` 和 `debts.json` 中记录。消费时从信用账户扣款，同时在负债中增加相应金额。
>
> **自动创建**：`credit` 类型账户无需手动添加，在首次使用信用支付（如"花呗买衣服200"）时自动创建对应的 `credit` 类型账户和负债条目。

## 边界情况

- 账户不存在 → 询问是否添加
- 余额为负 → 提示透支
- 重复添加 → 提示已存在
