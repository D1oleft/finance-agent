# 定期收支

## 流程

### 1. 初始化检查
检查 `~/finance/data/recurring.json` 是否存在，不存在则创建。

### 2. 操作类型

**查看**：
```
🔄 定期收支

支出：
- 房租：2,000元/月（每月1号，银行卡）
- 网费：100元/月（每月5号，支付宝）

收入：
- 工资：8,000元/月（每月15号，银行卡）

每月净收入：5,900元
```

**添加**：
```
/recurring 添加 房租 每月1号 2000 支出 居住 银行卡
→ 已添加定期支出：房租 2,000元/月
```

**删除**：
```
/recurring 删除 rec_001
→ 已删除：房租
```

**暂停**：
```
/recurring 暂停 rec_001
→ 已暂停：房租（不会自动记录）
```

### 3. 自动执行
自动执行依赖用户打开对话时检查。当用户打开对话时，系统检查是否有到期的定期收支需要执行：
1. 创建交易记录
2. 更新账户余额
3. 推送提醒

> **注意**：定期收支无法在后台自动运行。只有当用户主动打开对话时，才会检查并执行到期的定期收支。如果错过了执行日期，会在下次打开对话时补执行。

## 边界情况

- 日期无效（如2月30号） → 提示并建议最近的有效日期
- 账户不存在 → 询问是否创建
- 重复添加 → 提示已存在
