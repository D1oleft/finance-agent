# 账单导入

## 流程

### 1. 识别来源
- 用户发送文件 → 读取文件
- 用户说"导入支付宝" → 提示发送文件
- 用户说"导入微信" → 提示发送文件

### 2. 解析CSV

**支付宝格式**（通常前几行为说明信息，数据从包含"交易时间"的行开始）：
```
交易时间,交易分类,交易对方,商品说明,收/支,金额,收/付款方式,交易状态
2026-06-16 12:30:00,餐饮美食,肯德基,午饭,支出,25.00,花呗,交易成功
```

> **注意**：支付宝CSV导出文件通常包含文件头说明行（如"支付宝交易记录明细"等），解析时需跳过这些行，找到以"交易时间"开头的表头行开始解析。金额字段可能带有人民币符号"¥"，需去除后再解析。

**微信格式**（通常前16行为说明信息，数据从第17行开始）：
```
交易时间,交易类型,交易对方,商品,收/支,金额(元),支付方式,当前状态
2026-06-16 12:30:00,商户消费,肯德基,午饭,支出,25.00,零钱,支付成功
```

> **注意**：微信CSV导出文件前16行为说明信息，需跳过。金额字段带有"¥"符号，需去除。交易时间格式可能与标准格式不同，需做兼容处理。

**智能识别**：如果格式不匹配，尝试按列解析。优先识别包含"交易时间"的行作为表头。

### 4. 去重
比对：时间 + 金额 + 描述，相同则跳过。

### 5. 确认导入
```
📥 账单导入

平台：支付宝
时间范围：2026-05-01 ~ 2026-05-31
交易笔数：128笔
总支出：5,680元
总收入：8,200元

确认导入？（回复"确认"或"取消"）
```

### 6. 导入完成

导入完成后，自动同步更新 `accounts.json` 中各账户余额：根据导入的交易记录，按账户汇总收支，更新对应账户的余额。

```
✅ 导入完成

已导入：128笔
跳过重复：3笔

账户余额已同步更新：
- 支付宝：+2,500元
- 微信：-1,200元
- 银行卡：+800元
```

## 边界情况

- 文件格式错误 → 提示格式要求
- 无交易记录 → 提示文件为空
- 重复导入 → 自动去重
