# 提醒设置

## 流程

### 1. 初始化检查
检查 `~/finance/data/reminders.json` 是否存在，不存在则执行统一初始化流程（参见 SKILL.md）。

### 2. 操作类型

**设置提醒**：
```
/remind 设置 花呗还款 每月9号
→ 已设置提醒：花呗还款（每月9号）
```

**查看提醒**：
```
🔔 提醒列表

- 花呗还款：每月9号（支付宝）
- 信用卡还款：每月25号（银行卡）
```

**删除提醒**：
```
/remind 删除 花呗还款
→ 已删除提醒：花呗还款
```

### 3. 提醒触发
提醒依赖用户打开对话时检查。当用户打开对话时，系统检查是否有到期提醒，如有则主动推送：
```
⏰ 提醒

明天是花呗还款日
建议今天还款，避免逾期
```

> **注意**：提醒功能无法在后台自动运行。只有当用户主动打开对话时，才会检查并推送到期的提醒。

## 日期转换规则

具体日期的自然语言转换：
- "6月25号" → "2026-06-25"（当前年份）
- "明天" → 当前日期 + 1 天
- "后天" → 当前日期 + 2 天
- "下周一" → 下一个星期一
- "月底" → 当前月最后一天
- "每月9号" → 保持 "每月9号" 格式（周期性提醒）

> 所有具体日期统一转换为 `YYYY-MM-DD` 格式存储。

## 边界情况

- 提醒不存在 → 提示
- 重复设置 → 提示已存在
- 日期无效 → 提示正确格式
