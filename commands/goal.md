# 存钱目标

## 流程

### 1. 初始化检查
检查 `~/finance/data/goals.json` 是否存在，不存在则创建。

### 2. 操作类型

**设置目标**：
```
/goal 设置 旅行基金 5000
/goal 设置 旅行基金 5000 2026-12-31
→ 已设置目标：旅行基金 5,000元，截止：2026-12-31
```

> **可选参数**：截止日期（deadline）为可选参数，格式为 `YYYY-MM-DD`。不指定则无截止日期。

**查看进度**：
```
🎯 旅行基金

目标：5,000元
已存：2,000元（40%）
剩余：3,000元
截止：2026-12-31

进度条：████████░░░░░░░░ 40%

按当前速度，预计10月完成
```

**存入**：
```
/goal 存入 旅行基金 500
→ 已存入：旅行基金 +500元
→ 已存：2,500元（50%）
🎉 已完成一半！
```

> **重要**：存入操作必须同时执行以下三步：
> 1. 更新 `goals.json` 中的 `saved` 金额
> 2. 在 `transactions.jsonl` 中写入一笔支出交易（type: `expense`，category: `转账`）
> 3. 更新 `accounts.json` 中对应账户的余额（扣除存入金额）

**查看所有目标**：
```
🎯 存钱目标

- 旅行基金：2,000/5,000元（40%）
- 新手机：1,000/3,000元（33%）
- 紧急基金：5,000/10,000元（50%）
```

## 边界情况

- 目标不存在 → 提示
- 存入金额超过剩余 → 提示并设置为完成
- 截止日期已过 → 提示延期或调整目标
